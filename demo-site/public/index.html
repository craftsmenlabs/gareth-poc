Why

“Validate your assumptions automically and make them part of your development proces”

How

“By describing assumptions in a structured language, we want automatically validate if the goals a product owner set on forehand are reached.”

What

“We want to create a DSL that product owners can use to describe their assumptions for a user story. We want to validate these described assumptions by adding gluecode and running these assumptions in a framework that is available next to your application in production.”
